# secondbrain-db plugin v2.0.0

**Tracks `secondbrain-db` CLI v2.0.0** (released today). The plugin
version moves in lockstep with the CLI it wraps ŌĆö easier for users to
pick a compatible pair.

## What v2 changed in the CLI

### Storage ŌĆö `data/` is gone

v1 wrote two aggregate files per entity on every `create`/`update`/`delete`:
- `data/<entity>/records.yaml` ŌĆö the full list of records
- `data/<entity>/.integrity.yaml` ŌĆö the full integrity manifest

These were *merge-hostile*: two parallel PRs adding two different
documents both rewrote the same files, producing git conflicts even when
the changes were semantically disjoint. This killed multi-agent /
batched-PR workflows.

v2 replaces both with **per-md sidecars**: each `<id>.md` has a sibling
`<id>.yaml` carrying its content/frontmatter/record SHAs and optional
HMAC signature. Two PRs adding two different docs touch disjoint files
and merge with zero conflict.

```
docs/notes/hello.md    ŌåÉ content + frontmatter
docs/notes/hello.yaml  ŌåÉ integrity sidecar
```

### Doctor ŌĆö default scope = working-tree changes only

`sbdb doctor check` / `fix` / `sign` now scan only files that differ from
`HEAD`. Pass `--all` for a full audit. Premise: committed history was
already verified, so re-scanning thousands of clean files on every
invocation is wasteful.

Outside a git repo, doctor falls back to `--all` automatically with a
stderr notice.

### Doctor ŌĆö new `migrate` subcommand

One-shot, idempotent v1 ŌåÆ v2 conversion:

```
sbdb doctor migrate
```

Reads each entity's legacy `data/<entity>/records.yaml` and
`.integrity.yaml`, writes a per-md `<id>.yaml` sidecar next to each
existing `<id>.md`, then removes `data/`. Safe to re-run.

### Init ŌĆö bare scaffold

`sbdb init --template <name>` is **gone**. `sbdb init` now produces only
`.sbdb.toml` + empty `schemas/` and `docs/` directories. Reference
schemas (`notes`, `adr`, `discussion`, `task`) ship with this plugin
under `skills/secondbrain-db/reference/schemas/` ŌĆö copy any of them as
a starting point.

### CLI surface unchanged

Every other command (`create`, `update`, `delete`, `get`, `list`,
`query`, `search`, `index`, `graph`, `events`, `schema`, `untracked`)
keeps the same flags and JSON output shape.

### NEW: embeddable Go library

CLI v2.0.0 ships a stable, semver-versioned Go library at
`github.com/sergio-bershadsky/secondbrain-db/pkg/sbdb` so other Go
applications can embed sbdb directly:

```go
import "github.com/sergio-bershadsky/secondbrain-db/pkg/sbdb"

db, _ := sbdb.Open(ctx, sbdb.Config{Root: "."})
defer db.Close()

doc, _ := db.Repo("notes").Create(ctx, sbdb.Doc{
    Frontmatter: map[string]any{"id": "hello", "created": "2026-04-28"},
    Content:     "# Hello",
})
```

The CLI is now itself a thin consumer of `pkg/sbdb`. Anything the CLI
does, your Go application can do.

## Plugin changes shipped here

### Hooks ŌĆö updated for v2 wire format

- **`integrity-check.py`** (PostToolUse): now parses the v2 `{"action":
  "doctor.check", "scope": ..., "drifts": [...]}` JSON and classifies
  drift causes as pure-drift (auto-fixable), tamper, bad-sig, or missing
  pair. v1 keys (`drift_count`, `tamper_count`) and exit codes (4, 6, 7)
  are no longer parsed.
- **`integrity-final-check.py`** (Stop): same wire-format update; uses
  `--all` for the final audit so the entire KB is verified before the
  agent stops.
- **`guard-docs.py`** (PreToolUse): also blocks direct edits to `.yaml`
  sidecar files under `docs/`. Block message updated to reference v2
  CLI verbs (`sbdb create / update / delete`) and the correct go install
  path. Removed mention of the legacy `sbdb doc <verb>` syntax.

### Commands ŌĆö updated wording

- `sbdb-init`: drops `--template` flag reference; documents the bare
  scaffold + reference-schema copy flow.
- `sbdb-doctor`: drops exit codes 4/6/7 (replaced by single non-zero
  exit + per-doc cause list); documents `--all` flag and `migrate`
  subcommand.
- `sbdb-query`: replaces "reads only records.yaml" with the v2 walker
  description (concurrent frontmatter parse).
- `sbdb-review`: rewrites exit-code interpretation for v2; documents
  drift causes (`content_sha mismatch`, `frontmatter_sha mismatch`,
  `record_sha mismatch`, `bad_sig`, `missing-sidecar`, `missing-md`).

### Skills ŌĆö v2-aware

- `skills/secondbrain-db/SKILL.md`: rewritten storage-layout section
  (per-md sidecars, no `data/`); doctor scope explanation; v1 ŌåÆ v2
  migration callout; new "Embedding sbdb as a Go library" section
  with code samples.
- `skills/secondbrain-db-edit/SKILL.md`: revised integrity recovery
  loop for v2 drift causes; never-edit-sidecars rule; updated decision
  tree.
- `skills/secondbrain-db/reference/schema-format.md`: v2 schema shape;
  marks `records_dir`/`partition`/`date_field` as deprecated; new
  per-doc sidecar reference.
- **NEW**: `skills/secondbrain-db/reference/schemas/` ŌĆö four reference
  schemas (`notes`, `adr`, `discussion`, `task`) relocated from the
  CLI repo root. Strip of v1 deprecated fields. Copy and adapt.

### Manifest

- `plugin.json` and `marketplace.json` versions bumped to `2.0.0` to
  match the CLI release.

## Compatibility

Requires `secondbrain-db` CLI **v2.0.0 or later**. Earlier CLI versions
have the v1 layout the plugin's hooks no longer understand.

Users on v1 KBs: install CLI v2.0.0 first, then run
`sbdb doctor migrate` once. After that, install or upgrade this plugin.
The migration is idempotent.

Users who must stay on the v1 layout temporarily: pin the plugin to
`1.3.1`.
