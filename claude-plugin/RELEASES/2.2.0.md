# secondbrain-db plugin v2.2.0

**Tracks `secondbrain-db` CLI v2.2.0** (released 2026-05-08).
Lockstep version bump plus one plugin-side fix.

## What v2.2.0 added on the CLI side

Two features landed in the CLI between v2.1.0 and v2.2.0:

- **PGP-based ACL with blinded recipients and transparent git filter**
  ([#45](https://github.com/sergio-bershadsky/secondbrain-db/pull/45)).
  Per-document encryption with PGP recipients; a transparent
  clean/smudge git filter handles the encryption boundary so working
  copies stay readable.
- **JSON Schema 2020-12 with `x-*` extensions**
  ([#47](https://github.com/sergio-bershadsky/secondbrain-db/pull/47)).
  Schema files now use the JSON Schema 2020-12 dialect; sbdb-specific
  fields (virtuals, integrity policy, etc.) moved to `x-*` extension
  keys. Old schemas can be migrated with `sbdb schema migrate`.

## Plugin changes shipped here

- `plugin.json`: `2.1.0` → `2.2.0`
- `marketplace.json` entry: `2.1.0` → `2.2.0`
- `hooks/guard-docs.py`: improve the deny message shown when a direct
  edit to `docs/` is blocked.
  - Now names the `secondbrain-db-edit` skill so the full workflow
    loads on the next turn instead of being inferred from the message.
  - Documents `sbdb update --content-file <path>` for body rewrites
    (previously only `--field` and `--content` were listed, both
    awkward for multi-line markdown).
  - Documents `sbdb create --input -` / `sbdb update --input -` for
    combined body + frontmatter edits via JSON on stdin.
  - Calls out that writing the new body to a path *outside* `docs/`
    and passing it through `--content-file` is the supported flow,
    not a workaround.

The hook still denies the same set of tool calls — only the
human-readable rejection text changed.

## Known follow-up

The skill reference docs (`skills/secondbrain-db/reference/schema-format.md`)
do not yet describe the JSON Schema 2020-12 layout or `x-*` extensions.
A docs pass is queued separately and is not blocking this release.

## Compatibility

Requires CLI **v2.0.0 or later** (unchanged from v2.1.0). Users on the
v1 layout install CLI v2.0.0+, run `sbdb doctor migrate` once, then
install or upgrade this plugin. Pin to plugin `1.3.1` to stay on the
v1 layout.
