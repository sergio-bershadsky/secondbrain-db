# secondbrain-db plugin v0.1.0

**Initial release.**

The first cut of the Claude Code plugin for `secondbrain-db`. Wraps the `sbdb` CLI with hooks, agents, skills, and slash commands so AI sessions can manage a markdown knowledge base safely.

## Hooks

| Hook | Event | Purpose |
|---|---|---|
| `integrity-check.py` | `PostToolUse` (Write\|Edit) | Runs `sbdb doctor check` after every edit to `.md` or `.yaml` files inside `docs/` or `data/`. If `sbdb` is installed and `.sbdb.toml` is present, surfaces drift / tamper warnings inline so the AI sees them immediately. Auto-suggests `sbdb doctor fix --recompute` for pure drift and explains tamper recovery options. |
| `integrity-final-check.py` | `Stop` | Final integrity sweep before the session ends. Catches anything the per-edit hook missed (e.g. multi-edit batch where only the last edit's hook fires). |
| `guard-docs.py` | `PreToolUse` (Write\|Edit\|MultiEdit\|NotebookEdit\|Bash) | Blocks direct AI edits to `docs/` in sbdb-managed repos. AI must use `sbdb create / update / delete` instead. Bash command scanner catches in-place mutations (`sed -i`, `rm`, `mv`, `cp`, redirects). Prints `go install` guide if `sbdb` is missing from `PATH` or `~/go/bin`. |

All hooks are no-ops when `.sbdb.toml` is absent at the repo root, so they don't interfere with non-sbdb projects.

## Skills

Skills providing guided sbdb operations — see the `skills/` directory.

## Commands

Slash commands wrapping common sbdb tasks — see the `commands/` directory.

## Compatibility

Requires the `sbdb` CLI on `PATH` or in `~/go/bin`. Install with:

```
go install github.com/sergio-bershadsky/secondbrain-db@latest
```

The plugin degrades gracefully when `sbdb` is missing — guard hooks still fire and print install instructions; integrity hooks become no-ops.
