# secondbrain-db plugin v0.2.0

**Audit Trail Release.** Companion to `secondbrain-db` v1.1, which introduces the append-only event log.

## What's new

### Hook: `guard-events.py` (PreToolUse)

A second guard hook protects the events log from out-of-band edits.

- **Live log**: blocks Write / Edit / MultiEdit / NotebookEdit / Bash mutations targeting `.sbdb/events/*.jsonl`. The AI must use `sbdb event append` instead.
- **Archive**: blocks all writes to `.sbdb/events/archive/**` outright. Only `sbdb doctor fix` may seal a month.
- **Bash scanner**: catches `sed -i`, `rm`, `mv`, `cp`, `tee`, redirects (`>`, `>>`) targeting events paths.
- **Install guidance**: prints `go install github.com/sergio-bershadsky/secondbrain-db@latest` when the `sbdb` CLI is missing.

The hook activates only when `.sbdb.toml` is present at the repo root, so it never interferes with non-sbdb projects.

### Coordinated with sbdb-db v1.1 events feature

The plugin now assumes the host CLI supports the `sbdb event` command family (`append`, `types`, `show`, `repair`, `rebuild-registry`) and the doctor's event-window check. Pinning the plugin to v0.2.0 implies a compatible CLI is installed.

If a user has the v0.2.0 plugin but an older CLI, the events guard still fires (it's a pure Python check that only inspects file paths), but events themselves won't be written by sbdb. Running `sbdb event types` will fail with "unknown command" until the CLI is upgraded — a clear signal to upgrade.

## Coverage of immutability invariants

With both guards active, an AI session in an sbdb-managed repo physically cannot:

- Edit content under `docs/` directly (must go through `sbdb create / update / delete`)
- Append, edit, or remove anything in `.sbdb/events/*.jsonl` (must go through `sbdb event append`)
- Touch any sealed archive in `.sbdb/events/archive/**` (only `sbdb doctor fix` writes there)

The events log gains its full immutability guarantee: append-only at the kernel level (POSIX `O_APPEND`), append-only at the tool level (CLI is the only writer), and append-only at the agent level (PreToolUse hook blocks bypass attempts).
