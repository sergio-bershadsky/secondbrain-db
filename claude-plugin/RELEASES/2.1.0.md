# secondbrain-db plugin v2.1.0

**Tracks `secondbrain-db` CLI v2.1.0** (released today). Pure version
bump — no behavioural changes in the plugin itself. The CLI gained the
embeddable Go library at `pkg/sbdb`; the plugin already documented this
surface in v2.0.0, so no doc updates were needed.

## What v2.1.0 added on the CLI side

The CLI promoted all domain code from `internal/` to a public,
semver-stable Go library at `pkg/sbdb`. External Go applications can
now embed sbdb directly:

```go
import "github.com/sergio-bershadsky/secondbrain-db/pkg/sbdb"

ctx := context.Background()
db, _ := sbdb.Open(ctx, sbdb.Config{Root: "."})
defer db.Close()

doc, _ := db.Repo("notes").Create(ctx, sbdb.Doc{
    Frontmatter: map[string]any{"id": "hello", "created": "2026-04-30"},
    Content:     "# Hello",
})
```

Public API: `Open`, `DB`, `Repo`, `Doc`, sentinel errors (`ErrNotFound`,
`ErrConflict`, `ErrUnknownEntity`, `ErrValidation`, `ErrSchemaInvalid`,
`ErrIntegrity`), functional options (`WithLogger`, `WithClock`,
`WithIntegrityKey`, `WithIntegrityKeyLoader`, `WithWalkWorkers`).

Sub-packages all public:
`pkg/sbdb/{schema,query,storage,integrity,virtuals,events,kg,semantic,config,version,document,untracked}`.

The CLI is now itself a thin consumer of `pkg/sbdb`.

## Plugin changes shipped here

- `plugin.json`: `2.0.0` → `2.1.0`
- `marketplace.json` entry: `2.0.0` → `2.1.0`

That's it. The hooks, commands, skills, and reference schemas are
unchanged from v2.0.0 — the wire format and CLI behaviour the plugin
depends on did not change between v2.0.0 and v2.1.0. The library is
purely additive.

## Compatibility

Requires CLI **v2.0.0 or later** (same as plugin v2.0.0). Users on the
v1 layout install CLI v2.0.0+, run `sbdb doctor migrate` once, then
install or upgrade this plugin.

Pin to plugin `1.3.1` to stay on the v1 layout.
