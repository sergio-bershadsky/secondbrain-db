# secondbrain-db plugin v1.3.1

**Major version alignment + content cleanup.** This release jumps the plugin
version from `0.2.1` to `1.3.1` so it matches the underlying `secondbrain-db`
CLI release. From now on the plugin's published version is identical to the
`sbdb` binary it wraps — easier to reason about for users picking compatible
combinations.

## Why the jump

Previous releases (`0.1.0`, `0.2.0`, `0.2.1`) tracked plugin-internal
versioning while `secondbrain-db` itself moved through `1.0.0` → `1.1.1` →
`1.2.x` → `1.3.x`. The skew made it hard to tell which plugin worked with
which CLI without consulting the release notes. From `1.3.1` forward, the
two move in lockstep.

## Companion CLI changes (`secondbrain-db` v1.2.0 → v1.3.1)

The CLI underwent a major redesign in v1.2.0 and a follow-up usability fix
in v1.3.x:

- **Events are no longer stored on disk.** The entire `.sbdb/events/`
  directory, monthly archival, S3 archive target, hash-chain integrity for
  events, and the `[events]` config section are gone. Events are projected
  from git history on demand by a new command:

  ```
  sbdb events emit <commit-from> [<commit-to>|latest]
  ```

  Workers pipe its JSONL output instead of tailing files.

- **Removed CLI subcommands** (no longer exist):
  - `sbdb event append`
  - `sbdb event show`
  - `sbdb event types`
  - `sbdb event repair`
  - `sbdb event rebuild-registry`

- **Added:** `sbdb --version` flag (alongside the existing `sbdb version`
  subcommand).

- **Fixed in v1.3.1:** `go install …@latest` builds now report the correct
  version via `runtime/debug.BuildInfo` instead of always printing `dev`.

## Plugin changes shipped here

- **Removed `guard-events.py`.** The hook protected `.sbdb/events/**` from
  out-of-band edits, but that directory no longer exists in v1.2+. Keeping
  the hook would have it fire on every Write/Edit for no purpose.
- **Updated `hooks/hooks.json`** to drop the `guard-events.py` registration.
  The PreToolUse chain now ships only `guard-docs.py`, which is still the
  right protection — agents must use `sbdb create / update / delete` rather
  than direct file edits.
- **`plugin.json` and `marketplace.json` versions** bumped to `1.3.1`.
  The marketplace entry was previously stale at `0.1.0` while the manifest
  said `0.2.1`; both are now aligned.

## What did NOT change

- `guard-docs.py` — byte-identical
- `integrity-check.py`, `integrity-final-check.py` — byte-identical
- All skills (`secondbrain-db`, `secondbrain-db-edit`)
- All commands (`sbdb-search`, `sbdb-query`, `sbdb-doctor`, `sbdb-graph`,
  `sbdb-review`, `sbdb-init`)
- Agents (`kb-curator`)

## Compatibility

Requires `secondbrain-db` CLI **v1.3.0 or later**. Earlier CLI versions still
have the on-disk events log this plugin no longer guards; functionally fine
but the integrity story for events is no longer covered by the plugin.

Users on earlier sbdb versions who want the full guard suite should pin to
plugin `0.2.1`.
